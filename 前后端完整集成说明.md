# 前后端时间粒度功能完整集成说明

## 问题分析

从用户截图发现，前端显示的拆解结果是固定层级（"年度目标"、"季度计划"、"月度任务"等），而不是根据实际时间范围动态生成的层级结构。

**根本原因**：
1. 后端 `DecomposeChain` 虽然生成了 `time_hierarchy`（包含完整的时间层级结构），但没有将其返回给前端
2. 前端的 `transformPlanSchemaToYearNode` 函数只能看到 `tasks` 和 `milestones`，无法获取时间层级信息
3. 前端转换逻辑按照固定模式将 milestones 转换为 months，tasks 转换为 weeks/days

#### 2.2 添加时间层级转换函数

**文件**: `frontend/src/utils/transformer.ts`

```typescript
/**
 * 将 time_hierarchy 转换为 YearNode 格式
 */
export function transformTimeHierarchyToYearNode(
  timeHierarchy: TimeHierarchy
): YearNode {
  // 递归转换时间层级节点
  function convertNode(node: TimeHierarchyNode): any {
    const baseNode = {
      title: node.title,
      description: `${node.level.toUpperCase()} - ${node.title}`,
      start_date: node.start_date,
      end_date: node.end_date,
      task_date: node.task_date,
      estimated_hours: node.estimated_hours,
      priority: 1,
      children: [],
    };

    // 根据层级设置类型
    if (node.level === 'day') return { ...baseNode, type: 'day' };
    else if (node.level === 'week') return { ...baseNode, type: 'week' };
    else if (node.level === 'month') return { ...baseNode, type: 'month' };
    else if (node.level === 'quarter') return { ...baseNode, type: 'month' };
    else return { ...baseNode, type: 'year' };
  }

  // 递归构建层级结构
  function buildHierarchy(nodes: TimeHierarchyNode[]): any[] {
    return nodes.map(node => {
      const converted = convertNode(node);
      if (node.children && node.children.length > 0) {
        converted.children = buildHierarchy(node.children);
      }
      return converted;
    });
  }

  const topLevelNodes = buildHierarchy(timeHierarchy.hierarchy);

  return {
    title: timeHierarchy.goal,
    description: `时间粒度: ${timeHierarchy.granularity.toUpperCase()}\n总天数: ${timeHierarchy.total_days}天`,
    type: 'year',
    start_date: timeHierarchy.start_date,
    end_date: timeHierarchy.end_date,
    estimated_hours: topLevelNodes.reduce((sum, node) => sum + (node.estimated_hours || 0), 0),
    children: topLevelNodes,
  };
}

/**
 * 转换 PlanSchema 为 YearNode（优先使用 time_hierarchy）
 */
export function transformPlanSchemaToYearNode(plan: PlanSchema): YearNode {
  // 优先使用 time_hierarchy
  if (plan.time_hierarchy) {
    return transformTimeHierarchyToYearNode(plan.time_hierarchy);
  }

  // 回退到旧的转换逻辑

}
```

## 数据流程

### 完整请求流程

```
用户输入
  ↓
前端 TaskForm
  - start_date: "2024-01-01"
  - end_date: "2024-12-31"
  ↓
后端 API (task_decomposer/app.py)
  ↓
Orchestrator.decompose_task()
  ↓
DecomposeChain.run()
  ↓
1. 判断时间粒度 → YEAR
2. 生成 time_hierarchy
   - 4个季度
   - 12个月
   - 52周
   - 262个工作日
3. 调用 AI 生成任务计划
4. 合并结果到 PlanSchema
  - tasks: AI生成的任务列表
  - milestones: AI生成的里程碑
  - time_hierarchy: 时间层级结构 ⭐
  ↓
返回前端
  ↓
transformPlanSchemaToYearNode()
  - 检测到 plan.time_hierarchy 存在
  - 调用 transformTimeHierarchyToYearNode()
  - 递归转换层级节点
  ↓
前端显示
  - 根据实际时间范围显示正确层级
```
