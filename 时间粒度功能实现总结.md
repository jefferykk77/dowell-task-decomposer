# 时间粒度功能实现总结

## 功能概述

已成功实现任务时间粒度的自动判断和层级拆解功能。系统可以根据任务的开始时间和结束时间，自动判断任务属于年度、季度、月度、周度或每日任务，并从最大粒度开始逐级拆解到天级别。

## 核心功能

### 1. 时间粒度自动判断

根据时间范围自动判断任务粒度：
- **年度任务**: 时间跨度 ≥ 365天
- **季度任务**: 时间跨度 ≥ 90天
- **月度任务**: 时间跨度 ≥ 30天
- **周度任务**: 时间跨度 ≥ 7天
- **日度任务**: 时间跨度 < 7天

### 2. 层级拆解结构

系统实现了完整的层级拆解，从最大粒度到最小粒度：

```
年度任务 (Year)
├── 季度任务 (Quarter)
│   ├── 月度任务 (Month)
│   │   ├── 周度任务 (Week)
│   │   │   ├── 日度任务 (Day)
│   │   │   ├── 日度任务 (Day)
│   │   │   └── ...
```

### 3. 智能工作日过滤

- 默认只包含工作日（周一到周五）
- 可自定义每周工作日
- 自动计算每天工作小时数

## API 使用示例

### 前端请求示例

```typescript
const request: DecomposeRequest = {
  goal: "开发一个任务管理小程序",
  context: "需要支持用户注册、任务创建、进度追踪等功能",
  constraints: [
    "预算有限，使用开源方案",
    "技术栈：React + TypeScript"
  ],
  start_date: "2024-01-01",
  end_date: "2024-12-31",
  work_hours_per_day: 8.0,
  work_days_per_week: [0, 1, 2, 3, 4], // 周一到周五
  enable_evaluation: true
};
```

### 后端处理流程

1. **接收请求**：解析时间参数
2. **判断粒度**：根据时间范围判断最大粒度
3. **生成层级**：从最大粒度逐级拆解到天
4. **AI增强**：将时间层级信息传递给AI，生成更精确的任务计划
5. **返回结果**：包含完整的层级任务结构

## 核心函数说明

### `determine_time_granularity(start_date, end_date)`

判断时间粒度：
- 输入：开始日期和结束日期
- 输出：TimeGranularity 枚举值

### `decompose_task_by_time(goal, start_date, end_date, ...)`

执行任务拆解：
- 输入：目标、时间范围、工作日配置
- 输出：包含层级结构的字典
  ```python
  {
    "goal": "目标描述",
    "start_date": "2024-01-01",
    "end_date": "2024-12-31",
    "granularity": TimeGranularity.YEAR,
    "total_days": 366,
    "hierarchy": [...]
  }
  ```

### `flatten_hierarchy(hierarchy)`

扁平化层级结构：
- 输入：层级结构字典
- 输出：扁平化的任务列表
