# 前端开发计划：AI 任务拆解工具 (Web 版)

根据您的需求，我将为您构建一个基于 React 的全中文前端应用，与现有的 Python 后端无缝对接。该应用将专注于直观展示 "年 -> 月 -> 周 -> 日" 的任务拆解链路。

## 1. 技术栈选择
- **核心框架**: React + TypeScript (使用 Vite 构建，快速且类型安全)
- **样式方案**: Tailwind CSS (现代化的原子类 CSS 框架，构建美观界面)
- **UI 组件**: 自研轻量级组件 + Lucide React (图标库)
- **数据请求**: Axios
- **动画效果**: Framer Motion (用于丝滑的层级展开/折叠动画)
- **日期处理**: date-fns

## 2. 核心功能模块

### 2.1 任务输入区 (Task Input)
- **任务标题**: 文本输入框 (如 "2025年精通Python")
- **年份选择**: 下拉选择 (默认当前年份)
- **拆解策略**: 单选按钮 (AI智能拆解 / 规则拆解)
    - *注：选择 AI 策略将调用硅基流动模型进行个性化生成*
- **生成按钮**: 点击触发后端 API

### 2.2 拆解链路可视化 (Decomposition View)
为了清晰展示四层链路，我将采用 **"级联卡片流"** 的设计：

1.  **年度根节点 (Year)**
    - 展示年度核心目标、起止时间、关键里程碑。
    - 视觉上作为顶层容器。

2.  **月度层级 (Month)**
    - 以 12 宫格或时间轴形式展示。
    - 每个月展示该月的主题目标。
    - **交互**: 点击某个月份，展开该月下的周计划。

3.  **周度层级 (Week)**
    - 在选中的月份内，展示包含的周 (通常4-5周)。
    - 展示该周的重点攻克方向。
    - **交互**: 点击某一周，展开该周的每日任务。

4.  **每日层级 (Day)**
    - 最终的叶子节点。
    - 列表展示当天的具体执行事项 (ToDo List 样式)。
    - 包含复选框 (模拟完成状态) 和工时预估。

### 2.3 数据处理层
- **适配器模式**: 后端返回的是扁平化的 `months`, `weeks`, `days` 列表。前端将实现一个 `DataTransformer`，根据日期范围自动将其组装成 `Year -> Month -> Week -> Day` 的树状结构，以便于 UI 渲染。

## 3. 开发步骤

1.  **环境初始化**: 创建 `frontend` 目录，初始化 Vite React 项目。
2.  **基础设施**: 配置 Tailwind CSS，封装 Axios 请求拦截器 (指向 `http://127.0.0.1:8000`)。
3.  **数据层开发**: 定义 TypeScript 类型接口，实现数据转树形结构的逻辑。
4.  **组件开发**:
    - 开发 `TaskForm` 组件。
    - 开发层级展示组件 (`YearView`, `MonthCard`, `WeekCard`, `DayItem`)。
5.  **整合与联调**: 将前端与您刚才启动的 Python 后端进行联调，验证真实数据流。
6.  **UI 美化**: 调整配色（采用清爽的效率蓝风格），优化中文排版，添加加载动画。

## 4. 预期交付物
- 一个完整的 `frontend` 文件夹，包含所有源码。
- 浏览器中可交互的 Web 页面，支持输入任务并查看完整的 AI 拆解结果。
